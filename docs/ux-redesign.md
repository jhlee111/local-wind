# UX 리디자인 설계 — 단일 타임라인 + 반응형 레이아웃 (M2.6)

> 2026-07-12, 사용자 피드백 기반. 데이터·파이프라인은 그대로 두고 레이아웃/인터랙션만 재설계.
> 벤치마크: windy.app (모바일 하단 시트, 팔레트 색칠 슬라이더), windy.com (타임라인 바).

## 진단 — 지금 무엇이 어긋나 있나

사용자 피드백 3건은 모두 하나의 구조적 문제에서 나온다: **"시간"을 조종하는 곳이 3군데인데 서로 모른다.**

| 컨트롤 | 시간 범위 | 조작 결과 |
|---|---|---|
| 맵 하단 슬라이더 | 0–18h (프레임 인덱스) | 지도 래스터만 바뀜 |
| 패널 차트 | 고정 창 (now−24h → +18h) | 조작 불가, 고정 |
| 패널 주간 테이블 | 8일 (3h) | 표시만, 클릭 무반응 |

그래서 ① 차트(1h·24h창)와 테이블(3h·8일)의 x축이 달라 "비슷한 정보 두 덩어리"로 읽히고, ② 어디를 조작해도 다른 뷰가 안 따라오고, ③ 슬라이더는 시간을 "f07" 같은 숫자로만 보여준다. 추가로 ④ 스팟 패널이 떠 있는 모달이라 모바일 비친화적, ⑤ 범례가 하단 중앙의 최고 요지를 차지(참조 정보인데 컨트롤 자리에 있음).

## 설계 원칙 (Decision D12–D15)

- **D12 — 단일 시간 상태.** 앱 전역에 `selectedTime(T)` 하나. 맵 프레임·타임라인 플레이헤드·테이블 하이라이트 컬럼·차트 커서가 전부 T의 뷰. 어느 것을 조작해도 나머지가 따라온다. (테이블 컬럼 클릭/탭 → T 변경 → 맵·차트 동기화 — 사용자가 제안한 바로 그 인터랙션)
- **D13 — 모바일은 모달이 아니라 하단 시트.** windy.app 패턴: 지도 위 어디를 탭해도 하단에 도킹된 시트가 갱신. 시트는 3단계 — peek(스팟명+현재값 1줄) / half(주간 테이블 행들) / full(+상세 차트). 떠 있는 팝업 금지.
- **D14 — 타임라인이 곧 개요 차트.** 네이키드 `<input range>` 대신 커스텀 스크러버: 날짜 경계 + 요일 라벨 + **선택 스팟의 풍속을 팔레트로 색칠한 미니 스트립**(windy.app 슬라이더의 그라데이션 패턴). 슬라이더 자체가 "이번 주 언제 부나"의 개요가 된다. 시간 라벨은 "Sun 2 PM" 로컬 표기.
- **D15 — 범례는 참조물, 컨트롤 자리에서 빼기.** 하단 중앙 → 우하단 코너의 소형 바(축소/펼침 가능)로 강등. 모델·런 정보("HRRR 16Z f00")는 주 라벨에서 빼고 부가 정보 줄로 (D10 정직성은 유지하되 위계만 낮춤).

## 타깃 레이아웃

### 데스크톱 (≥ 768px)

```
┌────────────────────────────────────────────────────────────┐
│  MAP (fullscreen)                        ┌───────────────┐ │
│                                          │ SPOT CARD     │ │
│                                          │  헤더(현재값)  │ │
│                                          │  상세 차트     │ │  ← 차트 창 = T가 속한 날
│                                          │  주간 테이블   │ │  ← 컬럼 클릭 → T
│                                          └───────────────┘ │
│                                              [범례▁▂▃] ← 코너│
│ ┌────────────────────────────────────────────────────────┐ │
│ │ TIMELINE  SUN 13 ─ MON 14 ─ TUE 15 ─ … (8일)           │ │
│ │ ▓▓▒▒░░▒▓▓▓… ← 스팟 풍속 팔레트 스트립  ▼T "Sun 2 PM"    │ │
│ └────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘
```

- 타임라인은 **8일 전체**를 커버. 단, 맵 래스터가 존재하는 0–18h 구간을 시각적으로 구분(밝은 트랙)하고, T가 그 밖이면 지도는 마지막 프레임 유지 + "지도는 +18h까지" 뱃지. (래스터 범위 확장은 별개 백로그)
- 차트 창은 고정 24h 창이 아니라 **T가 속한 로컬 하루(±여백)**. 테이블 날짜 헤더 클릭 = 그 날로 차트 점프. 과거 날짜면 관측 오버레이가 자동으로 실린다 (오늘 이전 = 검증 뷰가 되는 부수효과).

### 모바일 (< 768px) — windy.app 패턴

```
┌──────────────────┐
│                  │
│   MAP            │  ← 시트 peek 높이만큼만 가려짐
│                  │
│        [범례칩]   │  ← 탭하면 펼침
├──────────────────┤
│ ⌃ Cabrillo · 12kt @250° now      │ ← peek (드래그 핸들)
│ SUN13 MON14 TUE15 …              │ ┐
│ kt   [4][8][6][5][2][3]…  →스크롤 │ │ half: 테이블 = 시간 내비게이션
│ gust [4][10][9][6]…              │ ┘   (탭 → T, 지도 동기화)
│ dir  [↗][→][→]…                  │
│ ── 상세 차트 (full로 펼치면) ──    │
└──────────────────┘
```

- **모바일에선 별도 슬라이더 없음** — 테이블 가로 스크롤+탭이 시간 내비게이션이고 now/T 컬럼 링이 플레이헤드. 시간 컨트롤러가 진짜로 하나가 된다.
- 시트 상태 전환: 핸들 드래그 or 헤더 탭. 지도 탭(임의 지점) → 시트 내용 교체(좌표 모드).
- 임의 지점도 같은 시트 — 주간 테이블은 Open-Meteo 클라이언트 fetch로 채움(기존 TODO와 결합, "source: Open-Meteo" 라벨로 D10 유지). 이러면 스팟/임의지점의 UI가 완전히 동일해져 "테이블이 사라졌다" 혼동 자체가 소멸.

## 구현 마일스톤 (작은 단위, 각각 배포 가능)

| # | 내용 | DoD |
|---|---|---|
| **UX-1** | 시간 상태 리팩터: `selectedTime` 단일 스토어(pub/sub, vanilla TS) — 맵/슬라이더/차트/테이블을 구독자로 전환. 시각 변화 없음 | 기존 동작 회귀 없음 (기능 동일) |
| **UX-2** | 데스크톱 타임라인 바(D14): 날짜 틱+팔레트 스트립+로컬시간 라벨, 범례 코너 이동(D15) | 슬라이더로 T 변경 시 테이블 하이라이트·차트 커서 동기 |
| **UX-3** | 테이블↔차트 연동(D12): 컬럼 탭→T, 날짜 헤더 탭→차트 그 날로, 차트 T-커서 | 테이블에서 수요일 오후 탭 → 차트가 수요일 상세로 |
| **UX-4** | 모바일 하단 시트(D13): 3단계 시트, <768px에서 플로팅 패널 대체, 시트 테이블=시간 내비 | iPhone 뷰포트(375px)에서 전 기능 사용 가능 |
| **UX-5** | 임의 지점 Open-Meteo 주간 테이블 + 마감 폴리시 (이 단계에서 frontend-design 스킬 로드해 시각 완성도 정리) | 스팟/임의지점 UI 동일, 라이트하우스 모바일 무리 없음 |

리스크/메모: 시트 드래그는 라이브러리 없이 CSS scroll-snap 또는 ~50줄 포인터 핸들러로(의존성 억제). 타임라인 팔레트 스트립 데이터는 이미 있는 spots_series 재사용(기본 스팟=Cabrillo, 임의지점 선택 시 그 지점 시리즈). 접근성: 커스텀 스크러버에 ARIA slider 롤 + 키보드 ←/→, 테이블 셀은 button화.

## 하지 않기로 한 것

- 데스크톱에서도 하단 시트로 통일 — 화면이 넓은데 지도를 가릴 이유 없음. 데스크톱=우측 카드, 모바일=시트로 분기.
- 차트/테이블 축 통일을 "차트를 8일로 늘리기"로 해결 — 1h 디테일이 뭉개짐. 축을 통일하는 게 아니라 **연동**하는 것이 정답 (master-detail).
